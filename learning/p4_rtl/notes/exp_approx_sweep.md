# exp 近似扫参笔记（模块 A / Python）

> 由 `scripts/exp_approx.py` 生成。对应学习计划：范围规约 + PWL，用真实 attention score 定误差预算与 Q 格式。

## 算法

$$
\exp(x)=2^{x\log_2 e}=2^{y_i+y_f}=2^{y_i}\cdot\mathrm{PWL}(2^{y_f}),\quad y_f\in[0,1)
$$

PWL 在段端点精确匹配 $2^{y_f}$。误差预算：**相对误差 $< 0.001$**，统计域为 $x\in[-8,0]$ 且 $\exp(x)\ge 0.0001$。

## 真实输入分布

从 P1 online softmax 数据流采样 `scores - m_new`（$P$ 路径）与 `m - m_new`（$\alpha$ 路径）；batch=1, heads=4, seq=512, $d$=64, block=64, causal。共 $539648$ 个有限样本。

| 统计 | 值 |
|------|-----|
| min | $-10.3216$ |
| p1 | $-5.4236$ |
| p5 | $-4.5437$ |
| median | $-2.5952$ |
| p95 | $-0.5077$ |
| p99 | $0.0000$ |
| max | $0.0000$ |
| 比例 $x>-8$ | $1.0000$ |
| 比例 $x>-16$ | $1.0000$ |
| 比例 $x>-32$ | $1.0000$ |

主质量几乎全在 $[-8,0]$（本采样 min=$-10.32$）。更负尾部对 softmax 权重可忽略，故不纳入相对误差验收。

## 误差拆分（浮点 PWL vs 定点 I/O）

纯浮点 PWL（无量化）在 8 段时 max rel 已 $<10^{-3}$；W=16 输出（`UQ0.16`）在小 $\exp(x)$ 处量化台阶主导 max rel（约 $2\%$）。因此要满足 $<10^{-3}$，需 **更细的输出小数位**，输入亦不宜粗于 ~10 bit 小数。

| 段数 | 浮点 PWL max rel |
|------|------------------|
| 4 | $3.759e-03$ |
| 8 | $9.387e-04$ |
| 16 | $2.346e-04$ |
| 32 | $5.865e-05$ |

## 选定配置

- **段数**：16
- **输入**：`Q6.10`（width=16，范围 $[-32,31.999]$）
- **输出**：`UQ0.24`（width=24，LSB=$2^{-24}$）
- **总 I/O 位宽**：40
- **误差**（目标域）：max rel=$7.249e-04$，p99 rel=$6.856e-04$，mean rel=$2.658e-04$，max abs=$7.028e-04$，n=539622
- **预算**：通过 （$<0.001$）

### 选型理由

1. 在过预算配置中优先 **总 I/O 位宽更小**，其次更少 PWL 段。
2. `Q6.10` 仍为 16-bit 输入总线，范围 $[-32,32)$ 覆盖真实 min （$-10.32$）；比 `Q8.8` 多 2 bit 小数，压低输入量化误差。
3. `UQ0.24` 使小 $\exp(x)$ 的量化相对误差落入预算；比 `UQ0.16` 多 8 bit 小数是满足 $<10^{-3}$ 的关键。
4. 16 段 PWL：浮点算法误差已远低于量化误差，再增到 32 段收益有限。

## 对后续 RTL 的约束

- 流水线与 `exp_approx_fixed` / `exp_approx_chosen` 一致：量化为 `Q6.10` → $\times\log_2 e$ → 拆 $y_i/y_f$ → 16 段 PWL → 按 $y_i$ 算术移位 → 量化 `UQ0.24`。
- 对拍阈值：目标域相对误差 $< 0.001$（$|\mathrm{ref}|\ge 0.0001$）。
- $x<-8$ 允许饱和到输出最小值，不单独考核相对误差。
- 常量 $\log_2 e$ 可用定点乘法；PWL 斜率/截距可固化为 ROM/组合逻辑表。

- 全样本（含极负尾）同配置 max rel=$7.249e-04$（仅供参考，非验收）。
